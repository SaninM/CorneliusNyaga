Secure Student Portal UI

Secure Student Portal UI

Concept UI for a secure student portal with MFA, role-based dashboard and secure file uploads.

UI/UX Design Front-End Development Cybersecurity Principles

Details — Secure Student Portal

UI/UX Design

Focus: clean student workflows, accessible forms, clear MFA flows, progressive disclosure for security settings and role-based dashboards. Wireframes -> high-fidelity mockups -> responsive prototypes.

Front-End Development

Technologies & Practices:

  • HTML5 semantic structure; ARIA where necessary for accessibility.
  • CSS methodologies (BEM / utility-first) and responsive design using CSS Grid & Flexbox.
  • JavaScript: modular code, fetch-based API integration, optimistic UI updates, client-side validation.
  • State management for complex dashboards (lightweight stores or framework state).
  • Performance: code-splitting, image optimization, lazy-loading resources.
Cybersecurity Principles

Core security principles applied to this project:

  • Least Privilege: Role-based access controls; users see only allowed data and actions.
  • Defense in Depth: Multiple layers — input validation (client + server), Web Application Firewall, secure session handling.
  • Authentication & MFA: Strong password policies, optional/required multi-factor authentication (TOTP/email/sms), secure account recovery flows.
  • Secure Data Handling: Use HTTPS everywhere, encrypt sensitive data at rest and in transit, avoid storing sensitive tokens in localStorage.
  • CSRF & XSS Mitigations: CSRF tokens for state-changing requests, output encoding, Content Security Policy (CSP).
  • Audit & Logging: Tamper-evident logs, admin audit trails, event alerts for suspicious activity.
  • Secure File Uploads: Validate file types and sizes server-side, store uploads outside webroot, generate scanned previews.

Nairobi Streets - A Visual Study

Nairobi Streets - A Visual Study

Photography series documenting everyday life, color, and movement on the streets of Nairobi.

Composition Storytelling Mobile Photography

Details — Nairobi Streets

Composition

Composition is the deliberate arrangement of visual elements to create a cohesive image. Key concepts applied:

  • Rule of Thirds: Place subjects on grid intersections to create balance.
  • Leading Lines: Use streets, fences, and pathways to guide the viewer’s eye.
  • Framing: Use doorways, windows or arches to frame subjects for depth.
  • Foreground / Midground / Background: Layer elements to create depth and context.
  • Negative Space & Minimalism: Let areas of calm support strong subjects.

Applied examples: candid portraits framed by market stalls, motion blur for moving vehicles while keeping a sharp subject.

Storytelling — "An Urban Morning Adventure"

Story excerpt:

"Dawn broke over Nairobi, the market waking in a slow, familiar rhythm. I followed Amina as she carried crates of mangoes through a maze of alleys — every face told a new beginning. The light hit like a promise; a rickshaw rattled past, scattering dust that turned golden. That single walk captured vendors trading, children in uniforms on their way to school, and the city’s quiet resilience. Each frame in the series aims to be a chapter in that morning's journey."

This narrative approach guides sequencing: opening shots (context), mid-series portraits (character), closing images (reflection).

Mobile Photography

Tips and workflow for mobile captures: shoot in RAW (where possible), lock exposure, use manual focus for portraits, bracket exposures for backlit scenes, and edit nondestructively in Lightroom Mobile or Snapseed.

Secure E‑Commerce Checkout

Secure E‑Commerce Checkout

Checkout flow designed for conversions while enforcing payment-layer security and PCI-conscious patterns.

Payment UX Backend Integration Security Hardening

Details — E‑Commerce Checkout

Payment UX

Design patterns: single-page checkout, inline validation, clear error states, progress indicators, saved payment options with explicit consent, and accessibility for keyboard users.

Backend Integration

Best practices: server-side payment tokenization, idempotent order endpoints, retry logic, webhook verification and secure secrets management (KeyVault/env vars).
